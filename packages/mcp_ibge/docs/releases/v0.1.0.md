# mcp-ibge v0.1.0 — Release Notes

**Model Context Protocol server for Brazilian IBGE public data.**
First public release, focused on a stable **Localidades** server.

## Highlights

- 🗺️ **7 stable Localidades tools** — regions, states, municipalities and
  districts, with codes and hierarchy resolved, plus fuzzy/accent-insensitive
  municipality search and name → IBGE code resolution.
- 📦 **Typed, traceable responses** — every tool returns
  `{"metadata": {...}, "data": ...}` with `source_name`, `source_url`,
  `endpoint`, `params` and `retrieved_at`, so agents (and humans) can verify
  every answer against the official source.
- 🔒 **Security-hardened by default** — no shell execution, no local file
  access, no arbitrary URLs, requests restricted to
  `servicodados.ibge.gov.br`, input validation on every parameter, request
  timeouts and a response size limit. See [docs/security.md](../security.md).
- ⚡ **In-memory TTL cache** to reduce repeated calls to the IBGE API.
- 🧪 **Full test suite** (`pytest` + `respx`, no network access required) and
  **CI** on every push/PR (ruff, ruff format, pytest, optional mypy).
- 🧭 **Agregados/SIDRA tools and a population indicator included as
  experimental previews** — list aggregates, inspect metadata/variables/
  periods/locations, query data, and get a municipality's estimated
  population. They work and are tested, but their interface may still change.

## What's included

### Stable: Localidades

| Tool | What it does |
| --- | --- |
| `listar_regioes` | List Brazil's 5 geographic regions |
| `listar_estados` | List all 26 states + the Federal District |
| `obter_estado` | Get a state by abbreviation (`"RJ"`) or IBGE code (`"33"`) |
| `listar_municipios` | List the municipalities of a state |
| `buscar_municipio` | Fuzzy search municipalities by name |
| `obter_codigo_municipio` | Resolve a municipality name + UF to its 7-digit IBGE code |
| `obter_municipio_por_codigo` | Get municipality details by IBGE code |
| `listar_distritos` | List the districts of a municipality |

### Experimental: Agregados/SIDRA & indicators

`listar_agregados`, `obter_metadados_agregado`, `listar_variaveis_agregado`,
`listar_periodos_agregado`, `listar_localidades_agregado`,
`consultar_agregado`, and `consultar_populacao_municipio`, plus the
`comparar_municipios` prompt and the `ibge://status` resource.


## Security & robustness notes

- All outbound requests are restricted to `https://servicodados.ibge.gov.br`
  (validated at startup — the server refuses to start with any other domain).
- All tool inputs (UF, municipality code, aggregate ID, periods, territorial
  levels) are validated before any network call.
- Timeouts (`MCP_IBGE_TIMEOUT`) and a response size limit
  (`MCP_IBGE_MAX_RESPONSE_SIZE_BYTES`) protect against slow or oversized
  responses.
- Errors are translated into informative messages without leaking stack
  traces to the MCP client; logs always go to `stderr`, never `stdout`.

Full details in [docs/security.md](../security.md).

## Known limitations

- Agregados/SIDRA tools and `consultar_populacao_municipio` are
  **experimental**: parameters and output shape may change in a future
  release before being marked stable.
- `streamable-http` transport is for local/trusted use; not yet hardened for
  public/remote deployments.
- This server depends on `servicodados.ibge.gov.br`; outages, schema
  changes, or aggregates discontinued/renamed after a new Census can affect
  responses.
